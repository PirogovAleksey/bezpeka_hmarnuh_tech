Тема 9.4: Виклики та ризики AI в безпеці

Adversarial AI — Атаки

  • Reconnaissance: AI-powered scanning
  • Phishing: LLM-generated emails
  • Deepfakes: Voice/video social engineering
  • Malware: AI-assisted evasive malware

Adversarial AI — На ML

  • Evasion: Модифікація input для bypass
  • Poisoning: Corrupting training data
  • Model stealing: Extraction через queries
  • Prompt injection: Маніпуляція LLM

Explainability (XAI)

  • Чому модель вважає це malicious?
  • Analyst потребує пояснення
  • Compliance та audit вимоги
  • Tools: LIME, SHAP, Attention
  • Black box не acceptable

Data Challenges

  • Insufficient: Мало прикладів rare attacks
  • Imbalanced: 99.9% normal, 0.1% malicious
  • Label quality: Wrong labels = bad model
  • Concept drift: Patterns змінюються
  • Bias: Historical bias in data

Red Flags у AI Security

  • "100% detection" — нереалістично
  • "No false positives" — неможливо
  • No explainability — pure black box
  • "Set and forget" — потребує maintenance
  • Vague "AI-powered" — без деталей

Майбутнє AI в Security

  • Autonomous SOC: Мінімальний human intervention
  • Generative AI: LLMs як standard tools
  • Federated Learning: Privacy-preserving ML
  • AI vs AI: Defensive vs offensive arms race

Ключові висновки

  • AI — інструмент, не magic
  • Anomaly detection — killer app
  • Human-in-the-loop essential
  • Data quality критична
  • Adversarial threats реальні
  • Explainability matters
