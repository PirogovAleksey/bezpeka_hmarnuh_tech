Тема 9.1: Основи AI/ML для безпеки

Чому AI/ML в Security?

  • Volume: Терабайти логів щодня
  • Velocity: Тисячі змін на годину
  • Variety: Сотні типів ресурсів
  • Shortage: 3.5+ млн дефіцит спеціалістів
  • Rule-based detection не масштабується

Ключові терміни

  • AI: Системи, що імітують людський інтелект
  • ML: Навчання з даних без явного програмування
  • Deep Learning: Нейронні мережі з багатьма шарами
  • Generative AI: Створення контенту (GPT-4, Claude)

Типи Machine Learning

  • Supervised: Labeled data → classification (malware/benign)
  • Unsupervised: Unlabeled data → anomaly detection
  • Reinforcement: Trial-and-error → adaptive policies

Metrics для Security ML

  • Precision: True positives / all positive predictions
  • Recall: Detected threats / all actual threats
  • F1 Score: Balance of precision та recall
  • FPR: False Positive Rate — критичний для alert fatigue

Переваги ML над Rules

  • Виявлення novel/zero-day attacks
  • Адаптація до нових patterns
  • Обробка великих обсягів даних
  • Зменшення manual analysis
  • Continuous improvement з новими даними
