Тема 9.2: AI для виявлення загроз

Anomaly Detection

  • Навчити модель "нормальної" поведінки
  • Виявляти відхилення від норми
  • Point: Spike in traffic
  • Contextual: Login at 3 AM
  • Collective: Slow data exfiltration

Алгоритми Anomaly Detection

  • Isolation Forest: Швидкий, high-dimensional
  • One-Class SVM: Класичний підхід
  • Autoencoders: Neural network, reconstruction error
  • LSTM: Для time-series sequences

UEBA — User Behavior Analytics

  • Login patterns: Час, location, device
  • Data access: Що, скільки, коли
  • Network: Connections, data transfer
  • Use cases: Insider threat, compromised account, privilege abuse

Network Traffic Analysis

  • DDoS: Abnormal traffic patterns
  • Lateral movement: Unusual east-west traffic
  • C2 communication: Command-and-control detection
  • Exfiltration: Unusual outbound flows
  • Challenge: Encrypted traffic, dynamic IPs

ML-based Malware Detection

  • Static: File structure, strings, imports
  • Dynamic: Behavior при виконанні
  • Hybrid: Static + Dynamic
  • Переваги: Zero-day, polymorphic, faster detection

Cloud ML Services

  • AWS GuardDuty: ML threat detection
  • Amazon Macie: Sensitive data в S3
  • Azure Sentinel: UEBA, Fusion correlation
  • Google Chronicle: Security analytics
