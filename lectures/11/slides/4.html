Тема 11.4: Уроки та Incident Response

Спільні патерни інцидентів

  • IAM misconfiguration: Overly permissive roles
  • Missing detection: Months undetected
  • Credential exposure: Hardcoded secrets
  • Supply chain trust: Blind trust in vendors
  • Patch management: Slow response

Ключові уроки

  • Least Privilege: Завжди мінімальні права
  • Defense in Depth: Багато шарів захисту
  • Assume Breach: Планувати compromise
  • Visibility: Logging everything
  • Speed: Швидка detection та response

Cloud IR Framework

  • 1. Preparation: Playbooks, tools, training
  • 2. Detection: SIEM, alerts, anomalies
  • 3. Containment: Isolate, revoke access
  • 4. Eradication: Remove threat, patch
  • 5. Recovery: Restore, validate
  • 6. Lessons: Post-mortem, improve

Cloud-Specific IR Actions

  • Credentials: Rotate all compromised keys
  • IAM: Disable suspicious users/roles
  • Network: Security group isolation
  • Snapshots: Preserve evidence (EBS, disk)
  • Logs: Export CloudTrail, VPC Flow

IR Automation

  • AWS: Step Functions + Lambda
  • Azure: Logic Apps + Sentinel
  • SOAR: Automated playbooks
  • Auto-isolate compromised instances
  • Auto-revoke leaked credentials

Ключові висновки

  • Misconfiguration — #1 причина breaches
  • Supply chain attacks зростають
  • Detection time критично важливий
  • IR planning до інциденту
  • Вчитися на чужих помилках
