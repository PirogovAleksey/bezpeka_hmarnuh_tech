Тема 11.2: Case Study — Capital One та SolarWinds

Capital One (2019) — Overview

  • 106 млн записів клієнтів
  • Атакувальник: колишній співробітник AWS
  • Вектор: SSRF через misconfigured WAF
  • Штраф: $80M + $190M settlement
  • Найбільший банківський breach того часу

Capital One — Attack Chain

1
WAF SSRF
2
IMDS
3
Credentials
4
S3 List
5
Exfiltrate

Capital One — Lessons

  • IMDSv2: Require session tokens
  • Least Privilege: Не давати list all buckets
  • WAF hardening: Обмежити outbound
  • Detection: Alert на unusual API calls
  • Network segmentation: Обмежити metadata access

SolarWinds (2020) — Overview

  • APT29 (Cozy Bear) — Russian SVR
  • 18,000+ організацій отримали malware
  • US Treasury, Commerce, DHS compromised
  • Undetected 9+ months
  • Найбільша supply chain атака в історії

SolarWinds — Attack Chain

  • 1. Compromise SolarWinds build system
  • 2. Inject SUNBURST backdoor
  • 3. Signed legitimate update
  • 4. Customers deploy malicious update
  • 5. C2 через DNS, lateral movement

SolarWinds — Lessons

  • Software supply chain: Verify integrity
  • Build pipeline security: Isolated, monitored
  • Zero Trust: Don't trust internal software
  • SBOM: Software Bill of Materials
  • Detection: Behavioral analysis, not just signatures
