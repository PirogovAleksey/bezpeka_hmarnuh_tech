Тема 10.2: Криптографія та шифрування

Симетричне шифрування

  • Один ключ для encryption та decryption
  • AES-256: Стандарт, рекомендований
  • ChaCha20: Альтернатива, швидший
  • Переваги: Швидкість, ефективність
  • Недоліки: Key distribution problem

Асиметричне шифрування

  • Пара: public key + private key
  • RSA: Класичний (2048, 4096 bit)
  • ECDSA/ECDH: Менші ключі, та сама безпека
  • Переваги: Вирішує key distribution
  • Недоліки: Повільний, не для великих даних

Encryption At-Rest

  • Захист даних на storage
  • Рівні: Storage, volume, database, application
  • AWS: S3 SSE, EBS encryption
  • Azure: Storage Service Encryption
  • GCP: Default encryption всіх даних

Encryption In-Transit

  • Захист даних під час передачі
  • TLS 1.2+: Мінімальна версія
  • TLS 1.3: Рекомендована
  • Managed certificates: ACM, Key Vault
  • Internal traffic: mTLS, Service Mesh

Хеш-функції та підписи

  • SHA-256: Integrity verification
  • HMAC: Keyed hashing для auth
  • Digital Signatures: Authenticity + integrity
  • Hash + encrypt з private key = signature

Що НЕ захищає encryption

  • Compromised credentials
  • Application-level vulnerabilities
  • Authorized insider threats
  • Encryption — один шар, не панацея!
