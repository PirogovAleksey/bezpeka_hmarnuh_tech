Тема 10.4: HSM, Confidential Computing, Compliance

Hardware Security Modules (HSM)

  • Dedicated hardware для crypto operations
  • Keys ніколи не покидають HSM
  • FIPS 140-2 Level 3 certification
  • AWS CloudHSM, Azure Dedicated HSM, GCP HSM
  • Use cases: PKI, PCI compliance, code signing

Confidential Computing

  • Encryption In-Use — захист під час processing
  • Intel SGX: Secure Enclaves
  • AMD SEV: Encrypted Virtualization
  • Cloud: Azure Confidential VMs, GCP Confidential
  • Use: Multi-party computation, data collaboration

Data Loss Prevention (DLP)

  • Content inspection: PII, PCI, PHI detection
  • Context analysis: who, where, how
  • Policy enforcement: block, alert, encrypt
  • Google Cloud DLP, Amazon Macie, Microsoft Purview

Data Masking & Tokenization

  • Masking: Replace з fake but realistic
  • Static (copy) vs Dynamic (on-the-fly)
  • Tokenization: Replace з token, original in vault
  • No mathematical relationship
  • Often for PCI compliance

Compliance Requirements

  • GDPR: Encryption, pseudonymization, right to erasure
  • PCI DSS: Encrypt stored cards, TLS, HSM for keys
  • HIPAA: Encryption recommended, breach safe harbor
  • Data Residency: In-country storage requirements

Ключові висновки

  • Classification first — знати що захищати
  • Encryption everywhere: rest, transit, use
  • Key management критичний
  • Envelope encryption — стандарт cloud
  • HSM для high assurance
  • Defense in depth — encryption один шар
