Тема 10.3: Key Management

Чому Key Management критичний?

  • Компрометація ключа = компрометація всіх даних
  • Втрата ключа = втрата доступу до даних
  • Key rotation для long-term security
  • Encryption без key management — ілюзія безпеки

Cloud KMS Services

  • AWS KMS: Integration з 100+ сервісів, CloudTrail
  • Azure Key Vault: Keys, secrets, certificates
  • GCP Cloud KMS: Hierarchical, HSM support
  • Automatic key rotation available

Envelope Encryption

  • DEK: Data Encryption Key — шифрує дані
  • KEK: Key Encryption Key — шифрує DEK
  • Master key ніколи не покидає KMS
  • Efficient: symmetric для data, asymmetric для key
  • Стандартний pattern для cloud

Key Rotation

  • Automatic: AWS KMS annual rotation
  • Manual: Create new, re-encrypt, deprecate old
  • Rotate annually minimum
  • Immediate при suspected compromise
  • Automate where possible

BYOK vs HYOK

  • BYOK: Import свого ключа в cloud KMS
  • Контроль над generation, backup
  • HYOK: Ключ залишається on-premises
  • External Key Manager
  • Максимальний контроль, складніший management

Best Practices

  • Use managed KMS services
  • Enable automatic rotation
  • Separate keys per environment/purpose
  • Audit all key operations
  • Test key recovery procedures
