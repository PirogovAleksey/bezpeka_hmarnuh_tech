Тема 10.1: Основи захисту даних у хмарі

Чому захист даних особливий у хмарі?

  • Multi-tenancy: Спільна інфраструктура
  • Резидентність даних (Data residency): Різні юрисдикції
  • Provider access: Технічна можливість доступу
  • Network exposure: Публічні мережі
  • Відповідність (Compliance): GDPR, HIPAA, PCI DSS

Класифікація даних

  • Public: Для вільного поширення
  • Internal: Внутрішня інформація
  • Confidential: Бізнес-конфіденційна
  • Restricted: Найвищий захист (PII, фінанси)

Типи Sensitive Data

  • PII: Ім'я, email, телефон, адреса
  • PHI: Медичні записи (HIPAA)
  • PCI: Номери карток, CVV
  • Credentials: Паролі, API keys, tokens
  • IP: Код, патенти, trade secrets

Data Lifecycle

Спільна відповідальність (Shared Responsibility) для даних

  • Provider: Physical security, infrastructure encryption
  • Customer: Data classification, encryption, access
  • Shared: Управління ключами (Key management)
  • Customer завжди відповідальний за свої дані!

Три стани даних

  • У стані спокою (At Rest): На диску, в storage
  • При передачі (In Transit): Під час передачі мережею
  • У використанні (In Use): В пам'яті, під час обробки
  • Кожен стан потребує окремого захисту
