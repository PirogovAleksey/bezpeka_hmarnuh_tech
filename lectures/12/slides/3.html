Тема 12.3: Програма безпеки та DevSecOps

Security Program Layers

  • Governance: Policies, standards, roles
  • Controls: Preventive, detective, corrective
  • Operations: Monitoring, response, maintenance
  • Framework: NIST CSF, CIS, CSA CCM
  • Continuous improvement cycle

Team Structure

  • Centralized: Dedicated team, consistency
  • Federated: Champions, scalability
  • Hybrid: Central + embedded (recommended)
  • Roles: Architect, Engineer, Analyst, DevSecOps
  • Security Champions Program

Key Metrics

  • MTTD: Mean Time to Detect (<24h)
  • MTTR: Mean Time to Remediate (<4h critical)
  • Coverage: % assets monitored
  • Compliance: % resources compliant
  • Automation: % auto-remediated

DevSecOps Pipeline

  • Pre-commit: IDE plugins, hooks
  • Build: SAST, SCA, secrets, IaC scan
  • Test: DAST, container scan, API tests
  • Deploy: Policy gates, validation
  • Runtime: CWPP, CSPM, detection

DevSecOps Tools

  • SAST: SonarQube, Semgrep, Checkmarx
  • SCA: Snyk, Dependabot, OWASP DC
  • Secrets: GitLeaks, TruffleHog
  • IaC: Checkov, tfsec, KICS
  • Container: Trivy, Grype, Clair
