Тема 12.2: Стратегії міграції в хмару

6 Rs of Migration

  • Rehost: Lift-and-shift без змін
  • Replatform: Lift з оптимізацією
  • Repurchase: Перехід на SaaS
  • Refactor: Re-architect для cloud-native
  • Retain: Залишити on-premises
  • Retire: Вивести з експлуатації

Security Migration Framework

  • 1. Assessment: Inventory, classification, gaps
  • 2. Planning: Architecture, IAM, network
  • 3. Foundation: Landing zone, baseline
  • 4. Migration: Waves з validation
  • 5. Optimization: Continuous improvement

Landing Zone Components

  • Account structure: Multi-account strategy
  • Network: VPCs, segmentation, connectivity
  • IAM: Roles, policies, federation
  • Logging: Centralized audit, SIEM
  • Guardrails: Preventive та detective controls

AWS Control Tower

  • Multi-account з Organizations

  • Service Control Policies (SCPs)

Azure Landing Zones

  • Azure Policy для governance
