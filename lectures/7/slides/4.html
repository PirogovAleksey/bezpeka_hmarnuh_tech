Тема 7.4: Інструменти та практика CSPM

Native Cloud Tools

  • AWS: Config, Security Hub, Trusted Advisor
  • Azure: Defender for Cloud, Azure Policy
  • GCP: Security Command Center
  • Глибока інтеграція, включені у вартість
  • Обмеження: single-cloud only

Third-Party Leaders

  • Prisma Cloud: Comprehensive, multi-cloud, Gartner leader
  • Wiz: Attack path analysis, graph-based, $10B+ startup
  • Orca: SideScanning, agentless
  • Lacework: Behavior-based anomaly detection

Open Source Options

  • Prowler: AWS/Azure/GCP, CIS Benchmarks, CLI
  • CloudSploit: Multi-cloud scanning
  • Steampipe: SQL interface до cloud APIs
  • Checkov: IaC scanning (Terraform, K8s)

Порівняння підходів

  • Native: Безкоштовно, single-cloud, базовий функціонал
  • Third-Party: $$$, multi-cloud, advanced features
  • Open Source: Безкоштовно, потребує налаштування
  • Вибір залежить від масштабу та бюджету

Стратегія впровадження

  • Фаза 1: Assessment — inventory, requirements
  • Фаза 2: Pilot — POC у non-prod
  • Фаза 3: Rollout — production, workflows
  • Фаза 4: Optimization — tuning, automation

Best Practices

  • Visibility first: Inventory перед enforcement
  • Prioritize: Critical + public exposure + production
  • Enable DevOps: Self-service, clear guidance
  • Automate carefully: Low-risk спочатку
  • Measure: MTTD, MTTR, compliance score

CSPM → CNAPP

  • CNAPP = CSPM + CWPP + CIEM + DSPM
  • Correlated findings: повна картина ризику
  • Single pane of glass
  • Gartner: 75% нових CSPM буде CNAPP до 2025

Ключові висновки

  • Misconfigurations — #1 причина хмарних інцидентів
  • CSPM автоматизує detection та remediation
  • Вибір tool: native → third-party → open source
  • Інтеграція з DevOps критична для успіху
