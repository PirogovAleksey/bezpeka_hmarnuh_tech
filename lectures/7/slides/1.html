Тема 7.1: Проблема Cloud Misconfiguration

Що таке CSPM?

  • Cloud Security Posture Management — управління станом безпеки хмари
  • Автоматичне виявлення misconfigurations
  • Моніторинг compliance у реальному часі
  • Пріоритезація та remediation ризиків

Чому CSPM критично важливий?

  • 65-70% хмарних інцидентів — через misconfigurations
  • 197 днів — середній час виявлення без CSPM
  • Сотні сервісів, тисячі ресурсів — неможливо перевірити вручну
  • Gartner: 60% enterprise використовуватимуть CSPM до 2025

Типові misconfigurations

  • Storage: Публічні S3 buckets, Azure Blob без автентифікації
  • Network: Security groups 0.0.0.0/0, відкритий SSH/RDP
  • Identity: Надмірні IAM permissions, без MFA
  • Encryption: Незашифровані диски, бази без TLS
  • Logging: Вимкнений CloudTrail, відсутні audit logs

Чому помилки трапляються?

  • Складність: AWS 200+ сервісів, кожен з десятками параметрів
  • Швидкість DevOps: Deploy без security review
  • IaC: Infrastructure as Code масштабує і помилки теж
  • Розрив навичок: DevOps ≠ Security expertise

Capital One Breach (2019)

  • Причина: Misconfigured WAF → SSRF-атака
  • Наслідки: 100+ млн записів клієнтів
  • Штраф: $80 млн від OCC
  • Root cause: Надмірні IAM permissions для WAF role

Інші відомі кейси

  • Twitch (2021): Misconfigured server → 125 GB source code
  • Facebook (2019): 540 млн записів у публічних S3
  • Microsoft Power Apps (2021): Публічні API, 38 млн записів
  • Спільне: Всі — misconfigurations, не zero-days

Еволюція підходів

  • Manual audits: Не масштабується, застаріває миттєво
  • Scripts: Важко підтримувати, неповне покриття
  • Native tools: Single-cloud, обмежена візуалізація
  • CSPM: Комплексне рішення для enterprise
