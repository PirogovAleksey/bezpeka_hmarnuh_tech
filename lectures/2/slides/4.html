Тема 2.4: Аудит та Compliance в хмарі

Cloud Auditor (NIST)

  • Один з 5 акторів архітектури NIST
  • Незалежна оцінка хмарних сервісів
  • Перевірка відповідності стандартам
  • Аудит безпеки, приватності, performance

Типи аудитів

Internal

Власними силами

External

Незалежні аудитори

Continuous

Автоматизований

Regulatory

Регуляторні перевірки

GDPR Requirements

  • Захист персональних даних громадян ЄС
  • Штрафи: до 4% обороту або €20M
  • Права суб'єктів: доступ, видалення, portability
  • Data Protection Officer (DPO)

SOC 2 Reports

  • Type I: design controls на дату
  • Type II: effectiveness за період
  • Trust Criteria: Security, Availability...
  • Запитуйте у провайдера!

Процес аудиту

1
Planning: scope, стандарти
2
Evidence: документи, логи
3
Testing: перевірка контролів
4
Reporting: findings

Right to Audit

  • Контрактне право аудиту провайдера
  • Прямий аудит: рідко дозволяють
  • Третя сторона: спільно обраний аудитор
  • Reports: SOC 2, ISO сертифікати

Shared Responsibility в аудиті

  • Провайдер: фізична безпека, гіпервізор
  • Клієнт: конфігурація, IAM, дані
  • Провайдер сертифікований ≠ ви compliant
  • Ваші workloads — ваша відповідальність

Compliance Automation

  • CSPM: автоматична перевірка конфігурацій
  • Policy-as-Code: OPA, Checkov
  • Cloud-native: AWS Security Hub, Azure Defender
  • Continuous compliance monitoring

Ключові тези

  • Compliance — це процес, не одноразова подія
  • Автоматизація критична для масштабу
  • Shared Responsibility — знайте свої обов'язки
  • Документуйте все для аудиторів
