Безпека хмарних технологій

Тема 2.1: Стандарти безпеки хмарних обчислень

Навіщо стандарти?

  • Спільна мова: клієнт і провайдер розуміють одне одного
  • Базовий рівень: мінімальні вимоги безпеки
  • Довіра: сертифікації підтверджують відповідність
  • Compliance: виконання регуляторних вимог

Ландшафт стандартів

  • Міжнародні: ISO/IEC 27017, 27018, 27001
  • Національні: NIST CSF, SP 800-53
  • Галузеві: PCI DSS, HIPAA, SOC 2
  • Консорціумні: CSA CCM, STAR

ISO/IEC 27017

  • Рекомендації з безпеки для хмарних сервісів
  • Розширює ISO 27002, +7 контролів
  • Чітко розділяє відповідальність
  • Для IaaS, PaaS, SaaS

ISO/IEC 27018

  • Захист персональних даних (PII) у хмарі
  • Важливо для GDPR compliance
  • Принципи: consent, transparency, minimization
  • Вимоги до провайдера щодо PII

NIST Cybersecurity Framework

  • Identify: активи, ризики, governance
  • Protect: safeguards
  • Detect: моніторинг, виявлення
  • Respond: реагування на інциденти
  • Recover: відновлення

CSA Cloud Controls Matrix

  • CCM v4: 197 контролів у 17 доменах
  • IAM, Data Security, Encryption, Logging...
  • CAIQ: анкета для оцінки провайдера
  • STAR: програма сертифікації

SOC 2

  • Аудиторський стандарт AICPA
  • Trust Criteria: Security, Availability, Confidentiality
  • Type I: на момент часу
  • Type II: за період (6-12 міс)

PCI DSS та HIPAA

  • PCI DSS: платіжні картки, 12 вимог
  • HIPAA: медичні дані (США)
  • Shared responsibility matrix
  • Специфічні вимоги для хмари

Як обрати стандарт?

  • Регуляторні вимоги: GDPR → ISO 27018
  • Галузь: фінанси → PCI DSS
  • Географія: USA → NIST, FedRAMP
  • Клієнти: що вимагають контракти?
