Тема 8.1: Від Point Solutions до CNAPP

Що таке CNAPP?

Проблема Tool Sprawl

  • CSPM: Configuration assessment
  • CWPP: Workload protection
  • CIEM: Identity management
  • Container Security: Image scanning
  • IaC Security: Terraform/CloudFormation
  • Кожен tool — окремі alerts без контексту

Еволюція Cloud Security

  • 2010-2015: Lift-and-shift, adapted on-prem tools
  • 2015-2019: Cloud-native, point solutions emerge
  • 2019-2021: Tool sprawl crisis, alert fatigue
  • 2021+: CNAPP consolidation

Чому point solutions недостатньо?

  • CSPM: "S3 bucket is public" — Medium
  • DSPM: "Contains PII" — High
  • CWPP: "Lambda has CVE" — Medium
  • Окремо: 3 findings серед тисяч
  • CNAPP: "Critical path to PII data"

Визначення CNAPP (Gartner)

  • Unified: Єдина платформа
  • Tightly integrated: Компоненти обмінюються контекстом
  • Dev + Production: Весь lifecycle
  • Cloud-native: Containers, serverless, IaC

Переваги CNAPP

  • Correlated context: Misconfiguration + vulnerability + permissions
  • Attack path analysis: Реальні ризики
  • Unified prioritization: Фокус на критичному
  • Single pane of glass: Одна консоль
