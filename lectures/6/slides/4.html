Тема 6.4: Застосування блокчейну в кібербезпеці

Незмінний Audit Log

  • Проблема: атакуючі видаляють логи
  • Рішення: хеші логів у блокчейні
  • Неможливо змінити без виявлення
  • Timestamping: доказ часу події

Варіанти Audit Log

PKI та Certificate Transparency

  • DigiNotar (2011): фейкові сертифікати
  • Централізовані CA — single point of failure
  • Blockchain PKI: децентралізована видача
  • Автоматичне Certificate Transparency

Software Supply Chain

  • SolarWinds (2020): malware в оновленнях
  • SBOM: Bill of Materials у блокчейні
  • Кожна збірка записується з хешем
  • Верифікація походження коду

in-toto + Blockchain

  • Кожен крок CI/CD підписується
  • Хеші зберігаються в блокчейні
  • Верифікація всього ланцюга
  • Захист від tampering

Threat Intelligence Sharing

  • Проблема: компанії не діляться
  • Рішення: анонімний обмін через блокчейн
  • Токени за якісну інформацію
  • PolySwarm: marketplace для threat detection

IoT Security

  • Mirai Botnet: атака через IoT
  • DID для пристроїв: унікальна ідентичність
  • Смарт-контракти: access control
  • IOTA: DAG для IoT

Обмеження блокчейну

  • Scalability: обмежена пропускна здатність
  • Privacy: public blockchains прозорі
  • Immutability: помилки не виправити
  • Complexity: складність інтеграції
