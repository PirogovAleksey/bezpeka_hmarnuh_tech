Тема 6.3: Смарт-контракти та їх безпека

Що таке смарт-контракт?

  • Програма на блокчейні
  • Code is Law: автоматичне виконання
  • Незмінність: після деплою не змінити
  • Trustless: не потрібен посередник

Платформи

  • Ethereum: Solidity, EVM, найбільша екосистема
  • Solana: Rust, висока швидкість
  • Hyperledger: Go/Java, enterprise
  • Cardano: Haskell, formal verification

Reentrancy Attack

  • Повторний вхід до функції
  • Стан не оновлений → повторне зняття
  • The DAO (2016): $60M втрачено
  • Захист: Checks-Effects-Interactions

Приклад Reentrancy

  • Погано: transfer → update balance
  • Добре: update balance → transfer
  • Спочатку змінюємо стан
  • Потім взаємодіємо із зовнішнім кодом

Integer Overflow

  • uint256 max + 1 = 0
  • До Solidity 0.8 — без перевірок
  • SafeMath: бібліотека захисту
  • Solidity 0.8+: автоматичні перевірки

Access Control

  • Відсутність onlyOwner
  • Публічні функції замість private
  • Parity Wallet: $30M через публічну функцію
  • Захист: OpenZeppelin AccessControl

Front-Running

  • Атакуючий бачить pending транзакцію
  • Відправляє свою з вищим gas
  • MEV: Miner Extractable Value
  • Flashbots: захист від MEV

Oracle Manipulation

  • Контракти не мають зовнішніх даних
  • Oracles надають ціни, дані
  • Flash Loan Attacks: маніпуляція цін
  • Chainlink: децентралізовані oracles

Security Tools

  • Slither: static analysis
  • Mythril: symbolic execution
  • Echidna: fuzzing
  • Certora: formal verification

Bug Bounty

  • Immunefi: найбільша Web3 платформа
  • Виплати до $10M за критичні баги
  • Постійний аудит community
  • Third-party audits: OpenZeppelin, Trail of Bits
