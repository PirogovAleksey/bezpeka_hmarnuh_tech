Тема 4.2: Безпека Docker

Контейнери vs VM

Механізми ізоляції Docker

Простори імен

Процеси, мережа, файлова система

Контрольні групи

CPU, пам'ять, введення-виведення

Union FS

Багатошарова файлова система

Усе це — механізми ядра Linux

Загроза: Вразливі образи

  • Застарілі пакети з відомими CVE
  • Шкідливе ПЗ у публічних образах
  • 2018: 17 шкідливих образів на Docker Hub
  • Захист: сканування (Trivy, Snyk)

Загроза: Втеча з контейнера

  • CVE-2019-5736 (runc): перезапис бінарника
  • CVE-2020-15257: доступ до файлової системи хоста
  • Спільне ядро = ризик втечі
  • Захист: оновлення, без привілейованого режиму

Загроза: Неправильна конфігурація

  • Привілейований режим: повні права root на хості
  • Docker-сокет: /var/run/docker.sock
  • Монтування системних каталогів: /, /etc, /var
  • Запуск від root всередині контейнера

Безпека образів

  • Мінімальні образи: Alpine, Distroless
  • Конкретні версії: не :latest
  • Сканування: Trivy, Docker Scout, Snyk
  • Підписування: Cosign, Docker Content Trust

Безпечний Dockerfile

  • Багатоетапна збірка: без інструментів збірки у фіналі
  • USER: непривілейований користувач
  • Не COPY секрети — залишаться в шарах образу
  • .dockerignore: виключіть зайве

Безпека середовища виконання (Runtime Security)

--cap-drop=ALL

Скинути всі привілеї

--read-only

ФС лише для читання

--no-new-privileges

Без ескалації

--memory, --cpus

Ліміти ресурсів

Управління секретами

  • НЕ в environment variables (для критичних)
  • НЕ в Dockerfile
  • Docker Secrets: для Swarm
  • Vault, AWS Secrets Manager: зовнішні
