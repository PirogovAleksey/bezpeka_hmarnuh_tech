Тема 4.3: Безпека Kubernetes

Архітектура Kubernetes

Загрози K8s

  • Exposed API Server: повний контроль
  • Compromised etcd: всі секрети
  • Pod misconfig: privileged, hostPID
  • RBAC misconfig: cluster-admin для всіх

RBAC: Role-Based Access Control

RBAC Best Practices

  • Уникайте wildcard (*) permissions
  • Не давайте cluster-admin без потреби
  • Окремі service accounts для кожного app
  • Регулярний аудит прав

Network Policies

  • Контроль трафіку між pods
  • Default deny: все заборонено
  • Явно дозволяйте потрібний трафік
  • CNI: Calico, Cilium, Weave

Pod Security Standards

  • Privileged: без обмежень (system)
  • Baseline: мінімальні обмеження
  • Restricted: суворі правила (production)
  • Застосовуються через namespace labels

Secrets Management

  • За замовчуванням — base64 (не шифровані!)
  • Encryption at rest: для etcd
  • External: Vault, Sealed Secrets
  • Не commit секрети в Git!

Admission Controllers

  • Перехоплюють запити до API
  • PodSecurity: enforce PSS
  • OPA Gatekeeper: policy-as-code
  • Kyverno: K8s-native policies

K8s Security Checklist

  • API Server — тільки приватна мережа
  • RBAC — мінімальні права
  • Network Policies — default deny
  • PSS: Restricted для production
