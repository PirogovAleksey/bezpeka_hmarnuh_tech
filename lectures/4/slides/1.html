Безпека хмарних технологій

Тема 4.1: Безпека віртуалізації

Що таке віртуалізація?

  • Кілька VM на одному фізичному сервері
  • Гіпервізор (VMM) — керує VM
  • Основа хмарних обчислень
  • Ізоляція workloads різних клієнтів

Типи гіпервізорів

VM Escape — найгірший сценарій

  • Атакуючий "виривається" з VM
  • Доступ до гіпервізора або інших VM
  • VENOM (2015): через floppy controller
  • Захист: патчі, мінімум virtual hardware

Side-Channel Attacks

  • Експлуатація спільних ресурсів (CPU cache)
  • Spectre/Meltdown (2018): читання пам'яті інших VM
  • L1TF: L1 cache leak
  • Захист: мікрокод, dedicated hosts

Hyperjacking

  • Шкідливий гіпервізор під легітимну ОС
  • Повний контроль без відома жертви
  • Blue Pill: концептуальний rootkit
  • Захист: Secure Boot, TPM, attestation

Noisy Neighbor

  • Одна VM споживає надто багато ресурсів
  • Вплив на інші VM на тому ж хості
  • CPU, Memory, I/O, Network contention
  • Захист: resource quotas, QoS

Захист гіпервізора

  • Мінімальна інсталяція: тільки потрібне
  • Окрема management network
  • MFA для management console
  • Регулярні патчі

Захист VM

  • Hardening: CIS Benchmarks
  • Encrypted VMs: диски зашифровані
  • Snapshot management: безпечне зберігання
  • VM templates: стандартизовані образи

Таблиця загроз

  • VM Escape: критичний — патчі
  • Side-Channel: високий — dedicated hosts
  • Hyperjacking: критичний — Secure Boot
  • Noisy Neighbor: середній — limits
