Тема 1.3: Моделі обслуговування — IaaS, PaaS, SaaS

Три моделі обслуговування

  • IaaS — Infrastructure as a Service
  • PaaS — Platform as a Service
  • SaaS — Software as a Service

IaaS — Infrastructure as a Service

  • Споживач отримує: VM, сховища, мережі
  • Контролює: ОС, додатки, дані
  • Приклади: AWS EC2, Azure VMs, GCE

IaaS — Безпекові зобов'язання

  • Патчинг операційних систем
  • Конфігурація Security Groups
  • Шифрування даних at rest
  • Моніторинг та логування
  • Управління доступом до VM

PaaS — Platform as a Service

  • Споживач отримує: платформу для додатків
  • Контролює: код додатків, конфігурації
  • Приклади: Heroku, App Engine, Azure App Service

PaaS — Безпекові зобов'язання

  • Безпека коду (OWASP Top 10)
  • Захист даних додатку
  • Управління API ключами
  • Secrets management

SaaS — Software as a Service

  • Споживач отримує: готовий додаток
  • Контролює: налаштування, користувачів
  • Приклади: Microsoft 365, Salesforce, Slack

SaaS — Безпекові зобов'язання

  • Управління ідентифікаціями (IAM)
  • Захист облікових записів (MFA)
  • Класифікація даних
  • Compliance з політиками

Порівняння моделей

  • IaaS: Максимальний контроль, найбільші зобов'язання
  • PaaS: Середній контроль, фокус на коді
  • SaaS: Мінімальний контроль, фокус на даних
