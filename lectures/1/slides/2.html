Тема 1.2: Еталонна архітектура NIST — 5 акторів

NIST SP 500-292

  • Cloud Computing Reference Architecture
  • Визначає 5 основних акторів
  • Описує їх ролі та взаємодію
  • Фундамент для розуміння хмарної безпеки

5 акторів еталонної архітектури

  • Cloud Consumer — споживач
  • Cloud Provider — провайдер
  • Cloud Auditor — аудитор
  • Cloud Broker — брокер
  • Cloud Carrier — носій

Cloud Consumer (Споживач)

  • Використовує хмарні сервіси
  • Відповідає за свої дані та додатки
  • Управляє доступом користувачів
  • Конфігурує безпекові налаштування

Cloud Provider (Провайдер)

  • Надає хмарні сервіси
  • Відповідає за фізичну безпеку
  • Забезпечує безпеку гіпервізорів
  • Підтримує compliance (SOC 2, ISO 27001)

Cloud Auditor (Аудитор)

  • Незалежна оцінка сервісів
  • Security audit — безпекові контролі
  • Privacy audit — конфіденційність
  • Performance audit — SLA перевірка

Cloud Broker (Брокер)

  • Посередник між споживачем і провайдером
  • Intermediation — покращення сервісів
  • Aggregation — об'єднання сервісів
  • Arbitrage — вибір провайдера

Cloud Carrier (Носій)

  • Транспортування хмарних сервісів
  • Забезпечує підключення
  • Шифрування каналів
  • Захист від DDoS
