Тема 5.1: Архітектура Serverless

Що таке Serverless?

  • Serverless ≠ "без серверів"
  • Провайдер керує інфраструктурою
  • Розробник фокусується на коді
  • Pay-per-execution модель оплати

Еволюція

  • Bare Metal: фізичні сервери
  • Віртуалізація: VM ізоляція
  • Контейнери: мікросервіси
  • Serverless: функції як сервіс

Function as a Service (FaaS)

  • Код виконується у відповідь на події
  • AWS Lambda (2014) — піонер
  • Azure Functions, Google Cloud Functions
  • Автоматичне масштабування 0 → ∞

Backend as a Service (BaaS)

  • Готові backend-компоненти
  • Firebase: auth, database, storage
  • Auth0: автентифікація
  • Stripe: платежі як сервіс

Event-Driven Architecture

Приклад тригерів AWS Lambda

  • API Gateway: REST/GraphQL endpoints
  • S3 Events: upload, delete
  • SQS/SNS: черги повідомлень
  • CloudWatch: scheduled cron jobs

Холодний старт (Cold Start)

  • Перший запуск функції — затримка
  • Завантаження runtime + код
  • Python/Node: ~100-200ms
  • Java/C#: 500ms - кілька секунд

Модель спільної відповідальності (Shared Responsibility)

  • Провайдер: hardware, OS, runtime, ізоляція
  • Клієнт: код, дані, IAM, конфігурація
  • Менше відповідальності ≠ менше ризиків
  • Нові вектори атак!

Переваги Serverless

  • Економія: платиш тільки за використання
  • Масштабування: автоматичне, миттєве
  • Швидкість: focus on business logic
  • Менше ops: no patching, no capacity planning

Обмеження

  • Таймаут: AWS Lambda max 15 хвилин
  • Пам'ять: до 10 GB
  • Stateless: немає persistent storage
  • Прив'язка до постачальника (Vendor Lock-in): складна міграція
