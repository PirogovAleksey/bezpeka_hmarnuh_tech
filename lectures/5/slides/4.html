Тема 5.4: Найкращі практики (Best Practices) безпеки Serverless

Принцип найменших привілеїв

Приклад IAM Policy

  • Погано: "s3:*" на "*"
  • Добре: "s3:GetObject" на конкретний bucket
  • Додавай Condition для IP, VPC
  • Використовуй Permission Boundaries

Input Validation

  • Всі events — потенційно шкідливі
  • HTTP, S3, SQS, DynamoDB Streams
  • Whitelist > Blacklist
  • Валідація типів, довжини, формату

Secrets Management

  • Ніколи не хардкодь credentials
  • AWS Secrets Manager — ротація
  • Parameter Store — SecureString
  • Кешуй secrets у warm container

Dependency Management

  • Lock файли: package-lock.json
  • SCA: Snyk, npm audit, pip-audit
  • Автоматичні PR для оновлень
  • Мінімізуй кількість залежностей

Logging та Monitoring

  • CloudWatch Logs — structured logging
  • X-Ray — distributed tracing
  • Алерти на аномалії
  • Не логуй sensitive data!

Network Security

  • Lambda у VPC для internal resources
  • VPC Endpoints — приватний доступ до AWS
  • Security Groups для Lambda
  • Private subnets для databases

API Gateway Security

  • Throttling: rate limits per client
  • WAF: OWASP rules
  • API Keys — для зовнішніх клієнтів
  • Cognito/Lambda Authorizers

Cost Protection

  • Concurrency limits — reserved/provisioned
  • Budget alerts — сповіщення про витрати
  • Dead Letter Queues — retry storms
  • Timeout на розумне значення

Security Tools

  • SAST: Semgrep, SonarQube
  • SCA: Snyk, Dependabot
  • DAST: OWASP ZAP
  • CSPM: Prowler, ScoutSuite

CI/CD Security

  • Security scanning у pipeline
  • Інфраструктура як код (Infrastructure as Code) — версіонування
  • Automated deployment — менше людських помилок
  • Immutable deployments
