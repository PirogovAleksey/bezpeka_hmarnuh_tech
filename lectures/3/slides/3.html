Тема 3.3: Мікросегментація мережі

Що таке мікросегментація?

  • Поділ мережі на маленькі ізольовані сегменти
  • Політики безпеки для кожного робочого навантаження
  • Аналогія: від відкритого офісу до кабінетів
  • На відміну від VLAN — рівень робочих навантажень

Навіщо мікросегментація?

Підходи до мікросегментації

На основі мережі

SDN, Security Groups

На основі хоста

iptables, агенти Illumio

На основі застосунків

Service Mesh (Istio)

Мікросегментація в хмарі: AWS

  • Security Groups: stateful firewall для EC2
  • Network ACLs: stateless для subnet
  • VPC Endpoints: приватний доступ до сервісів AWS
  • PrivateLink: приватне з'єднання між VPC

Мікросегментація в хмарі: Azure/GCP

  • Azure: NSG, ASG, Azure Firewall, Private Endpoints
  • GCP: VPC Firewall Rules, VPC Service Controls
  • Спільне: tags/labels для групування ресурсів

Kubernetes: Network Policies

  • Декларативне визначення дозволеного трафіку
  • podSelector: до яких pods застосовується
  • ingress/egress: вхідний/вихідний трафік
  • Заборона за замовчуванням (Default deny): все заборонено за замовчуванням

Service Mesh: Istio

  • mTLS: автоматичне шифрування між сервісами
  • AuthorizationPolicy: L7 контроль доступу
  • Управління трафіком: маршрутизація, повтори, тайм-аути
  • Спостережуваність: трасування, метрики, логування

Найкращі практики (Best Practices)

  • Заборона за замовчуванням (Default Deny): заборонено все, що не дозволено
  • Мінімальних привілеїв (Least Privilege): мінімально необхідний доступ
  • Моніторинг: журнали потоків, видимість
  • Автоматизація: інфраструктура як код

Типові помилки

  • Занадто широкі правила (0.0.0.0/0)
  • Ігнорування egress-трафіку
  • Відсутність моніторингу заблокованого трафіку
  • Ручне управління без автоматизації

Software-Defined Perimeter (SDP)

  • Ідея: зробити інфраструктуру невидимою для неавторизованих
  • Dark Cloud: порти невидимі — сканування нічого не знаходить
  • SPA: перший пакет містить криптографічну автентифікацію
  • Захист: від port scanning, DDoS, zero-day експлойтів

Архітектура SDP

Контролер

Автентифікує, авторизує, керує доступом

Initiating Host

Клієнт: автентифікується + SPA-пакет

Accepting Host

Сервер: default deny, відкриває порт після SPA

East-West vs North-South трафік

Чому East-West — головна загроза

  • Lateral movement: після проникнення атакуючий переміщується через E-W з'єднання
  • Неявна довіра: внутрішній трафік часто не інспектується
  • Обсяг: 70-80% трафіку — неможливо перевірити вручну
  • Рішення: мікросегментація + Service Mesh + mTLS
