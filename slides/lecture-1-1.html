Безпека хмарних технологій

Тема 1.1: Еволюція хмарних обчислень та визначення NIST

Еволюція обчислювальних моделей

  • 1960-1980: Мейнфрейми — централізовані обчислення
  • 1980-2000: Клієнт-сервер — розподілені мережі
  • 2000-2010: Веб-сервіси — інтернет-додатки
  • 2010+: Хмарні обчислення — віртуалізація, API

Ключові віхи розвитку хмари

  • 2006 — Amazon AWS (EC2, S3)
  • 2008 — Google App Engine
  • 2010 — Microsoft Azure
  • 2011 — NIST SP 800-145
  • 2014 — Kubernetes

Визначення за NIST SP 800-145

Модель для забезпечення зручного мережевого доступу на вимогу до спільного пулу налаштовуваних обчислювальних ресурсів, які можуть бути швидко надані та звільнені з мінімальними зусиллями.

5 характеристик хмарних обчислень

  • On-demand Self-service — самообслуговування
  • Broad Network Access — широкий доступ
  • Resource Pooling — пул ресурсів
  • Rapid Elasticity — еластичність
  • Measured Service — вимірюваність

On-demand Self-service

  • Самостійне замовлення ресурсів
  • Без участі персоналу провайдера
  • Безпека: Ризик створення незахищених ресурсів
  • Рішення: IAM політики, автоматизовані перевірки

Broad Network Access

  • Доступ з будь-якого пристрою
  • Стандартні протоколи (HTTPS, API)
  • Безпека: Збільшена поверхня атаки
  • Рішення: Шифрування, VPN, ZTNA

Resource Pooling (Multi-tenancy)

  • Спільні ресурси для багатьох клієнтів
  • Динамічне призначення
  • Безпека: Side-channel атаки, "noisy neighbor"
  • Рішення: Сильна ізоляція tenant'ів

Rapid Elasticity

  • Автоматичне масштабування
  • Відповідно до навантаження
  • Безпека: Нові точки входу при масштабуванні
  • Рішення: Автоматизовані Security Groups

Measured Service

  • Pay-as-you-go оплата
  • Моніторинг використання
  • Безпека: Виявлення аномалій (cryptojacking)
  • Ризик: Витік метаданих
