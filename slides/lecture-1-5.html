Тема 1.5: Моделі розгортання — Public, Private, Hybrid, Community

Чотири моделі розгортання NIST

  • Public Cloud — для всіх, спільна інфраструктура
  • Private Cloud — для однієї організації
  • Hybrid Cloud — комбінація public + private
  • Community Cloud — для спільноти зі спільними вимогами

Public Cloud

  • AWS, Azure, GCP — доступні будь-кому
  • Multi-tenancy — спільна інфраструктура
  • Переваги: економія, еластичність, глобальність
  • Ризики: менший контроль, data residency

Private Cloud

  • Виключно для однієї організації
  • Може бути on-premises або hosted
  • Переваги: повний контроль, compliance
  • Ризики: висока вартість, потребує експертизи

Private Cloud — реалізації

  • VMware vSphere — найпопулярніший enterprise
  • OpenStack — open-source
  • Azure Stack — Azure у вашому дата-центрі
  • AWS Outposts — AWS hardware on-premises

Hybrid Cloud

  • Інтеграція public + private
  • Workloads переміщуються між хмарами
  • Єдине управління та політики
  • Найскладніша модель для безпеки

Hybrid — типові сценарії

  • Cloud bursting — масштабування в public при піках
  • Data sovereignty — чутливі дані локально
  • Disaster Recovery — резерв в іншій хмарі
  • Модернізація — legacy on-prem, нове в cloud

Hybrid — безпекові виклики

  • Безпечне з'єднання між хмарами
  • Консистентні політики безпеки
  • Єдиний identity management
  • Централізований моніторинг
  • Data classification та DLP

Community Cloud

  • Спільнота зі спільними вимогами
  • AWS GovCloud — для уряду США
  • Healthcare clouds — HIPAA compliant
  • Financial clouds — PCI DSS compliant
  • Баланс між public та private

Multi-cloud — сучасний тренд

  • Кілька public провайдерів одночасно
  • Переваги: vendor lock-in, best-of-breed
  • Ризики: складність × кількість провайдерів
  • Потребує cloud-agnostic інструментів

Порівняння моделей

  • Public: економія, але менше контролю
  • Private: контроль, але дорого
  • Hybrid: гнучкість, але складність
  • Community: compliance, але обмежений вибір
