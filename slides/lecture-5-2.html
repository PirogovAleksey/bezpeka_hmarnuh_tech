Тема 5.2: Загрози та вразливості Serverless

Нова поверхня атаки

  • Традиційні вразливості залишаються
  • + Нові, специфічні для serverless
  • Більше точок входу (triggers)
  • Складніший ланцюг залежностей

OWASP Serverless Top 10

  • #1: Injection (SQL, NoSQL, Command)
  • #2: Broken Authentication
  • #3: Sensitive Data Exposure
  • #4: XML External Entities

OWASP Top 10 (продовження)

  • #5: Broken Access Control
  • #6: Security Misconfiguration
  • #7: XSS
  • #8: Insecure Deserialization

Event Injection

  • Шкідливі дані через event source
  • S3: filename з командами
  • SQS: payload injection
  • DynamoDB: NoSQL injection

Приклад: Command Injection

  • filename = event['filename']
  • os.system(f'convert {filename}')
  • Атака: "; rm -rf / #"
  • Захист: валідація, subprocess з list

Overprivileged Functions

  • Функція має більше прав ніж потрібно
  • "*" permissions — найгірша практика
  • Компрометація → доступ до всього
  • Least Privilege — мінімум прав

Insecure Dependencies

  • npm/pip packages з вразливостями
  • event-stream (2018): malicious code
  • Тисячі транзитивних залежностей
  • Захист: SCA (Snyk, Dependabot)

Secrets Exposure

  • Хардкодження credentials у коді
  • Environment variables у plain text
  • Логування sensitive data
  • Рішення: Secrets Manager, KMS

Denial of Wallet (DoW)

  • Атака на гаманець, не на сервіс
  • Генерація мільйонів invocations
  • $100,000+ за ніч — реальні кейси
  • Захист: rate limits, бюджетні alerts

Data Leakage через /tmp

  • /tmp зберігається між invocations
  • Container reuse — дані попередньої функції
  • Чутливі файли можуть залишитися
  • Захист: очищення, шифрування

Таблиця загроз

  • Injection: критичний — валідація
  • Overprivilege: високий — least privilege
  • Dependencies: високий — SCA
  • DoW: середній — rate limits
