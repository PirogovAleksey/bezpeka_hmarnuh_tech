Тема 4.2: Безпека Docker

Контейнери vs VM

  • VM: повна ОС, GB, хвилини запуску
  • Контейнери: спільне ядро, MB, секунди
  • Контейнери — слабша ізоляція!
  • Більша поверхня атаки

Механізми ізоляції Docker

  • Namespaces: процеси, мережа, файлова система
  • Cgroups: ліміти CPU, пам'яті, I/O
  • Union FS: layered filesystem
  • Усе це — механізми Linux kernel

Загроза: Вразливі образи

  • Застарілі пакети з CVE
  • Malware в публічних образах
  • 2018: 17 шкідливих образів на Docker Hub
  • Захист: сканування (Trivy, Snyk)

Загроза: Container Escape

  • CVE-2019-5736 (runc): перезапис бінарника
  • CVE-2020-15257: доступ до host filesystem
  • Спільне ядро = ризик escape
  • Захист: оновлення, без privileged

Загроза: Misconfiguration

  • Privileged mode: повні права root на хості
  • Docker socket: /var/run/docker.sock
  • Sensitive mounts: /, /etc, /var
  • Running as root всередині контейнера

Безпека образів

  • Мінімальні образи: Alpine, Distroless
  • Конкретні версії: не :latest
  • Сканування: Trivy, Docker Scout, Snyk
  • Підписування: Cosign, Docker Content Trust

Безпечний Dockerfile

  • Multi-stage builds: без build tools у фіналі
  • USER: non-root користувач
  • Не COPY секрети — залишаться в layers
  • .dockerignore: виключіть зайве

Runtime Security

  • --cap-drop=ALL: видаліть capabilities
  • --read-only: immutable filesystem
  • --no-new-privileges: без ескалації
  • --memory, --cpus: resource limits

Управління секретами

  • НЕ в environment variables (для критичних)
  • НЕ в Dockerfile
  • Docker Secrets: для Swarm
  • Vault, AWS Secrets Manager: зовнішні
