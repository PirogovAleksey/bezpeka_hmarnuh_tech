Тема 6.2: Децентралізоване управління ідентифікацією

Проблеми централізованої ідентифікації

  • Single Point of Failure: злам IdP = все
  • Data Breaches: мільярди записів
  • Privacy: IdP знає все про вас
  • Vendor Lock-in: залежність

Self-Sovereign Identity (SSI)

  • Ownership: ви володієте ідентичністю
  • Control: ви вирішуєте, що розкривати
  • Portability: не прив'язано до провайдера
  • Privacy: selective disclosure

Decentralized Identifiers (DIDs)

  • Стандарт W3C
  • did:method:identifier
  • did:ethr: Ethereum-based
  • did:ion: Bitcoin Layer 2

DID Document

  • id: сам DID
  • verificationMethod: публічні ключі
  • authentication: методи auth
  • service: endpoints

Verifiable Credentials (VCs)

  • Цифрові посвідчення з підписом
  • Issuer: видає (університет, держава)
  • Holder: зберігає у wallet
  • Verifier: перевіряє

Zero-Knowledge Proofs

  • Доведення без розкриття даних
  • Приклад: "Мені > 18" без дати народження
  • BBS+ Signatures: selective disclosure
  • zk-SNARKs: складні докази

EU Digital Identity Wallet

  • eIDAS 2.0 — кожен громадянин ЄС
  • Державні та приватні credentials
  • Обов'язкове для великих платформ
  • Запуск: 2024-2026

Microsoft ION

  • DID мережа на Bitcoin Layer 2
  • Тисячі операцій в 1 BTC транзакції
  • Інтеграція з Azure AD
  • Open source

SSI vs Традиційна ідентифікація

  • Традиційна: IdP контролює все
  • SSI: користувач контролює
  • Традиційна: centralized storage
  • SSI: децентралізоване зберігання
